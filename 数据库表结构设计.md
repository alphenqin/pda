# PDA系统数据库表结构设计

## 说明

本文档定义了PDA系统与WMS系统交互所需的数据库表结构。PDA通过写入数据库的方式与WMS系统进行数据交换。

---

## 1. 托盘表（pallet）

### 表名：`pallet`

### 用途
存储托盘基本信息，包括托盘编号、二维码/条码、型号、当前所在位置等。

### 字段说明

| 字段名 | 类型 | 长度 | 是否必填 | 主键 | 说明 | 示例值 |
|--------|------|------|----------|------|------|--------|
| id | BIGINT | - | 是 | 是 | 自增主键 | 1 |
| pallet_no | VARCHAR | 50 | 是 | 否 | 托盘编号，唯一标识 | "11-01" |
| barcode | VARCHAR | 200 | 是 | 否 | 托盘二维码或条码，唯一标识 | "PALLET-QR-CODE-11-01" |
| pallet_type | VARCHAR | 20 | 是 | 否 | 托盘型号：SMALL(小托盘) / LARGE(大托盘) | "SMALL" |
| swap_station | VARCHAR | 50 | 否 | 否 | 置换区站点，如 "1-SMALL", "2-LARGE" | "1-SMALL" |
| location_code | VARCHAR | 50 | 否 | 否 | 库位号，如 "2-01" | "2-01" |
| current_status | VARCHAR | 20 | 是 | 否 | 当前状态：EMPTY(空托) / OCCUPIED(占用) / IN_TRANSIT(运输中) | "OCCUPIED" |
| create_time | DATETIME | - | 是 | 否 | 创建时间 | 2025-01-15 14:30:00 |
| update_time | DATETIME | - | 是 | 否 | 更新时间 | 2025-01-15 14:30:00 |
| remark | VARCHAR | 500 | 否 | 否 | 备注信息 | - |

### 索引
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_pallet_no (pallet_no)`
- 唯一索引：`UNIQUE KEY uk_barcode (barcode)`
- 普通索引：`KEY idx_location_code (location_code)`
- 普通索引：`KEY idx_pallet_type (pallet_type)`

### SQL建表语句

```sql
CREATE TABLE `pallet` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `pallet_no` VARCHAR(50) NOT NULL COMMENT '托盘编号',
  `barcode` VARCHAR(200) NOT NULL COMMENT '托盘二维码或条码',
  `pallet_type` VARCHAR(20) NOT NULL COMMENT '托盘型号：SMALL/LARGE',
  `swap_station` VARCHAR(50) DEFAULT NULL COMMENT '置换区站点',
  `location_code` VARCHAR(50) DEFAULT NULL COMMENT '库位号',
  `current_status` VARCHAR(20) NOT NULL DEFAULT 'EMPTY' COMMENT '当前状态：EMPTY/OCCUPIED/IN_TRANSIT',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `remark` VARCHAR(500) DEFAULT NULL COMMENT '备注',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_pallet_no` (`pallet_no`),
  UNIQUE KEY `uk_barcode` (`barcode`),
  KEY `idx_location_code` (`location_code`),
  KEY `idx_pallet_type` (`pallet_type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='托盘表';
```

## 2. 阀门表（valve）

### 表名：`valve`

### 用途
存储阀门信息，包括阀门编号、型号、厂家、入库日期等，与托盘关联。

### 字段说明

| 字段名 | 类型 | 长度 | 是否必填 | 主键 | 说明 | 示例值 |
|--------|------|------|----------|------|------|--------|
| id | BIGINT | - | 是 | 是 | 自增主键 | 1 |
| valve_no | VARCHAR | 100 | 是 | 否 | 阀门编号，唯一标识 | "V20250101-001" |
| valve_model | VARCHAR | 100 | 是 | 否 | 阀门型号 | "DN50" |
| vendor_name | VARCHAR | 200 | 是 | 否 | 厂家名称 | "XX阀门厂" |
| inbound_date | DATE | - | 是 | 否 | 入库日期 | 2025-01-15 |
| pallet_no | VARCHAR | 50 | 是 | 否 | 托盘号，关联pallet表 | "11-01" |
| location_code | VARCHAR | 50 | 是 | 否 | 库位号 | "2-01" |
| valve_status | VARCHAR | 20 | 是 | 否 | 阀门状态：IN_STOCK(在库) / IN_INSPECTION(检测中) / INSPECTED(已检测) / OUTBOUND(已出库) | "IN_STOCK" |
| inspection_result | VARCHAR | 20 | 否 | 否 | 检测结果：PASS(合格) / FAIL(不合格) / PENDING(待检测) | "PENDING" |
| inspection_date | DATE | - | 否 | 否 | 检测日期 | - |
| outbound_date | DATE | - | 否 | 否 | 出库日期 | - |
| create_time | DATETIME | - | 是 | 否 | 创建时间 | 2025-01-15 14:30:00 |
| update_time | DATETIME | - | 是 | 否 | 更新时间 | 2025-01-15 14:30:00 |
| remark | VARCHAR | 500 | 否 | 否 | 备注信息 | - |

### 索引
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_valve_no (valve_no)`
- 普通索引：`KEY idx_pallet_no (pallet_no)`
- 普通索引：`KEY idx_location_code (location_code)`
- 普通索引：`KEY idx_vendor_name (vendor_name)`
- 普通索引：`KEY idx_valve_status (valve_status)`
- 组合索引：`KEY idx_vendor_model_date (vendor_name, valve_model, inbound_date)`

### SQL建表语句

```sql
CREATE TABLE `valve` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `valve_no` VARCHAR(100) NOT NULL COMMENT '阀门编号，唯一标识',
  `valve_model` VARCHAR(100) NOT NULL COMMENT '阀门型号',
  `vendor_name` VARCHAR(200) NOT NULL COMMENT '厂家名称',
  `inbound_date` DATE NOT NULL COMMENT '入库日期',
  `pallet_no` VARCHAR(50) NOT NULL COMMENT '托盘号',
  `location_code` VARCHAR(50) NOT NULL COMMENT '库位号',
  `valve_status` VARCHAR(20) NOT NULL DEFAULT 'IN_STOCK' COMMENT '阀门状态：IN_STOCK/IN_INSPECTION/INSPECTED/OUTBOUND',
  `inspection_result` VARCHAR(20) DEFAULT 'PENDING' COMMENT '检测结果：PASS/FAIL/PENDING',
  `inspection_date` DATE DEFAULT NULL COMMENT '检测日期',
  `outbound_date` DATE DEFAULT NULL COMMENT '出库日期',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `remark` VARCHAR(500) DEFAULT NULL COMMENT '备注',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_valve_no` (`valve_no`),
  KEY `idx_pallet_no` (`pallet_no`),
  KEY `idx_location_code` (`location_code`),
  KEY `idx_vendor_name` (`vendor_name`),
  KEY `idx_valve_status` (`valve_status`),
  KEY `idx_vendor_model_date` (`vendor_name`, `valve_model`, `inbound_date`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='阀门表';
```

## 3. 任务表（task）

### 表名：`task`

### 用途
存储AGV任务信息，包括入库、送检、回库、出库等任务。

### 字段说明

| 字段名 | 类型 | 长度 | 是否必填 | 主键 | 说明 | 示例值 |
|--------|------|------|----------|------|------|--------|
| id | BIGINT | - | 是 | 是 | 自增主键 | 1 |
| task_no | VARCHAR | 50 | 是 | 否 | 任务编号，唯一标识 | "R20250715145830999" |
| task_type | VARCHAR | 30 | 是 | 否 | 任务类型：INBOUND(入库) / SEND_INSPECTION(送检) / RETURN(回库) / OUTBOUND(出库) | "INBOUND" |
| status | VARCHAR | 20 | 是 | 否 | 任务状态：PENDING(待执行) / EXECUTING(执行中) / COMPLETED(已完成) / CANCELLED(已取消) / FAILED(失败) | "PENDING" |
| pallet_no | VARCHAR | 50 | 是 | 否 | 托盘号 | "11-01" |
| valve_no | VARCHAR | 100 | 否 | 否 | 阀门编号（可选） | "V20250101-001" |
| location_code | VARCHAR | 50 | 是 | 否 | 库位号 | "2-01" |
| from_station | VARCHAR | 100 | 是 | 否 | 起始站点 | "WAREHOUSE_SWAP_1" |
| to_station | VARCHAR | 100 | 是 | 否 | 目标站点 | "WAREHOUSE_LOCATION_2_01" |
| agv_task_id | VARCHAR | 100 | 否 | 否 | AGV系统返回的任务ID | - |
| create_time | DATETIME | - | 是 | 否 | 创建时间 | 2025-01-15 14:58:30 |
| start_time | DATETIME | - | 否 | 否 | 开始执行时间 | - |
| complete_time | DATETIME | - | 否 | 否 | 完成时间 | - |
| cancel_time | DATETIME | - | 否 | 否 | 取消时间 | - |
| operator | VARCHAR | 50 | 否 | 否 | 操作员 | - |
| error_message | VARCHAR | 500 | 否 | 否 | 错误信息 | - |
| remark | VARCHAR | 500 | 否 | 否 | 备注信息 | - |

### 索引
- 主键索引：`PRIMARY KEY (id)`
- 唯一索引：`UNIQUE KEY uk_task_no (task_no)`
- 普通索引：`KEY idx_task_type (task_type)`
- 普通索引：`KEY idx_status (status)`
- 普通索引：`KEY idx_pallet_no (pallet_no)`
- 普通索引：`KEY idx_valve_no (valve_no)`
- 普通索引：`KEY idx_create_time (create_time)`
- 组合索引：`KEY idx_type_status_time (task_type, status, create_time)`

### SQL建表语句

```sql
CREATE TABLE `task` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `task_no` VARCHAR(50) NOT NULL COMMENT '任务编号，唯一标识',
  `task_type` VARCHAR(30) NOT NULL COMMENT '任务类型：INBOUND/SEND_INSPECTION/RETURN/OUTBOUND',
  `status` VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT '任务状态：PENDING/EXECUTING/COMPLETED/CANCELLED/FAILED',
  `pallet_no` VARCHAR(50) NOT NULL COMMENT '托盘号',
  `valve_no` VARCHAR(100) DEFAULT NULL COMMENT '阀门编号（可选）',
  `location_code` VARCHAR(50) NOT NULL COMMENT '库位号',
  `from_station` VARCHAR(100) NOT NULL COMMENT '起始站点',
  `to_station` VARCHAR(100) NOT NULL COMMENT '目标站点',
  `agv_task_id` VARCHAR(100) DEFAULT NULL COMMENT 'AGV系统返回的任务ID',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `start_time` DATETIME DEFAULT NULL COMMENT '开始执行时间',
  `complete_time` DATETIME DEFAULT NULL COMMENT '完成时间',
  `cancel_time` DATETIME DEFAULT NULL COMMENT '取消时间',
  `operator` VARCHAR(50) DEFAULT NULL COMMENT '操作员',
  `error_message` VARCHAR(500) DEFAULT NULL COMMENT '错误信息',
  `remark` VARCHAR(500) DEFAULT NULL COMMENT '备注',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_task_no` (`task_no`),
  KEY `idx_task_type` (`task_type`),
  KEY `idx_status` (`status`),
  KEY `idx_pallet_no` (`pallet_no`),
  KEY `idx_valve_no` (`valve_no`),
  KEY `idx_create_time` (`create_time`),
  KEY `idx_type_status_time` (`task_type`, `status`, `create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务表';
```

## 表关系说明

### 主要关联关系

1. **valve.pallet_no** → **pallet.pallet_no**（阀门关联托盘）
2. **valve.location_code** → **location.location_code**（阀门关联库位）
3. **pallet.location_code** → **location.location_code**（托盘关联库位）
4. **task.pallet_no** → **pallet.pallet_no**（任务关联托盘）
5. **task.valve_no** → **valve.valve_no**（任务关联阀门）
6. **location.pallet_no** → **pallet.pallet_no**（库位关联托盘）

### 数据流转说明

1. **入库流程**：
   - PDA扫码托盘 → 查询/更新 `pallet` 表
   - PDA绑定阀门 → 插入 `valve` 表
   - PDA创建入库任务 → 插入 `task` 表
   - WMS读取任务 → 更新 `task.status` 和 `location.location_status`

2. **送检流程**：
   - PDA查询阀门 → 查询 `valve` 表
   - PDA创建送检任务 → 插入 `task` 表
   - WMS更新阀门状态 → 更新 `valve.valve_status = 'IN_INSPECTION'`

3. **回库流程**：
   - PDA查询阀门 → 查询 `valve` 表
   - PDA创建回库任务 → 插入 `task` 表
   - WMS更新阀门状态 → 更新 `valve.valve_status = 'IN_STOCK'`

4. **出库流程**：
   - PDA查询阀门 → 查询 `valve` 表
   - PDA创建出库任务 → 插入 `task` 表
   - WMS更新阀门状态 → 更新 `valve.valve_status = 'OUTBOUND'` 和 `valve.outbound_date`

## 字段值枚举说明

### pallet.pallet_type（托盘型号）
- `SMALL`：小托盘（1类托盘，共132个）
- `LARGE`：大托盘（2类托盘，共33个）

### pallet.current_status（托盘状态）
- `EMPTY`：空托盘
- `OCCUPIED`：已占用（有阀门）
- `IN_TRANSIT`：运输中

### valve.valve_status（阀门状态）
- `IN_STOCK`：在库
- `IN_INSPECTION`：检测中
- `INSPECTED`：已检测
- `OUTBOUND`：已出库

### valve.inspection_result（检测结果）
- `PENDING`：待检测
- `PASS`：合格
- `FAIL`：不合格

### task.task_type（任务类型）
- `INBOUND`：入库
- `SEND_INSPECTION`：送检
- `RETURN`：回库
- `OUTBOUND`：出库

### task.status（任务状态）
- `PENDING`：待执行
- `EXECUTING`：执行中
- `COMPLETED`：已完成
- `CANCELLED`：已取消
- `FAILED`：失败

### location.location_status（库位状态）
- `EMPTY`：空闲
- `OCCUPIED`：占用
- `RESERVED`：预留

## 注意事项

1. **字符集**：所有表使用 `utf8mb4` 字符集，支持中文和特殊字符
2. **时间字段**：使用 `DATETIME` 类型，自动记录创建和更新时间
3. **唯一性约束**：关键业务字段（如托盘号、阀门编号、任务编号）设置唯一索引
4. **索引优化**：根据查询需求建立合适的索引，提高查询性能
5. **外键约束**：建议在应用层维护数据一致性，不在数据库层面设置外键约束（提高性能）
6. **数据备份**：定期备份数据库，确保数据安全
7. **日志记录**：重要操作记录到 `operation_log` 表，便于问题排查
